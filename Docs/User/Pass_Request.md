# Pass Request

## Назначение
Этот документ объясняет, как человеку формулировать pass для агента внутри продуктового репозитория.

## Минимальная форма
1. Назвать текущий `ROAD-*` или указать новый pass внутри него.
2. Сформулировать узкую цель pass.
3. Перечислить область и документы-владельцы.
4. Перечислить ограничения и допустимые sync-поверхности.
5. Указать ожидаемый результат и проверки, если они не спорят с договорами репозитория.
