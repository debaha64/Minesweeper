# Pass Request

## Назначение
Этот документ объясняет, как человеку формулировать pass для агента внутри replicated product repo.

## Минимальная форма
1. Назвать текущий `ROAD-*` или указать новый pass внутри него.
2. Сформулировать узкую цель pass.
3. Перечислить scope и owner-documents.
4. Перечислить ограничения и допустимые sync-поверхности.
5. Указать desired outcome и проверки, если они не спорят с repo contracts.

- `../../Docs/User/*`, `../../Docs/Product/*`, `../../Docs/Technical/*`;
