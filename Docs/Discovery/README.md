# Discovery

`Docs/Discovery/*` хранит текущую истину продукта до перевода в требования и плановый контур.

## Владелец текущей истины
- `Interview.md` — владелец текущей аналитической истины созданного продукта.

## Минимум начального развёртывания раннего контура старта продукта
- `README.md` — карта аналитического слоя.
- `Interview.md` — текущее интервью продукта.

## Gate текущей истины
- интервью продукта, созданного начальным развёртыванием, стартует в состоянии `Статус_текущей_истины: Не_подтверждена`;
- начальный запрос пользователя не равен полному подтверждению текущей истины;
- пока пользователь не ответил явно, созданный репозиторий остаётся только в аналитическом контуре;
- агент не заполняет ответы интервью догадками;
- неявные предположения фиксируются отдельно как гипотезы;
- даже в аналитическом контуре первое записываемое действие допускается только после открытия рабочей ветки;
- блокирующие вопросы задаются пользователю до перехода дальше;
- bootstrap-заготовки не разрешают переход к `Docs/Product/*`, `Docs/Technical/*`, `Tools/*`, `Pipeline/*` и предметной реализации.

## Interview protocol
- владелец протокола интервью один: `Interview.md`;
- вопросы первого прохода собираются по классам `Контекст`, `Граница`, `Ограничение`, `Владение`, `Переход`;
- блокирующие вопросы задаются сразу;
- неблокирующие вопросы накапливаются для следующей фазы;
- delta-интервью всё равно использует тот же нумерованный формат с буквенными вариантами и рекомендуемым вариантом там, где это уместно.

## Ограничения, стек и зависимости
- ограничения первого прохода фиксируются отдельно от выбора стека;
- стек и зависимости подтверждаются только явным ответом пользователя, профилем продукта, требованием или отдельным техническим решением;
- стандартная библиотека Python считается зависимостью, если модуль зависит от системной среды;
- если запуск нельзя проверить, результат фиксируется как `не проверено`.

## Границы
- этот слой не дублирует `Plans/*` и `Logs/*`;
- history-fact изменений аналитического слоя закрывается через planning/log contour;
- `Discussion`, `Research` и `Requirements` не создаются до отдельного прохода, который явно открывает расширенный аналитический контур.
